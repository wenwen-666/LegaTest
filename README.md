# LegaTest — Intelligent Java Test Generation System

LegaTest provides an end-to-end workflow for Java unit-test generation, repair, and iterative coverage-driven evolution.

## Overview

```text
Dataset parser -> Test generator -> Test repair -> Iterative evolution
```

- `agents/dataset_parser.py`: scans Java projects under `dataset/` and produces class metadata such as `valid_classes.json`.
- `agents/test_generator`: generates initial JUnit test suites with LLM-assisted prompts and optional repair.
- `agents/test_repair`: repairs compilation/runtime failures with rule-based fixes and LLM repair.
- `agents/iterative_evolution`: evolves generated tests with coverage analysis, selection, crossover, mutation, and report management.

For module-specific details, see:

- `agents/test_generator/README.md`
- `agents/test_repair/README.md`
- `agents/iterative_evolution/README.md`

## Expected Layout

```text
<base_dir>/
  dataset/
    <project_name>/
      pom.xml
      src/main/java/...
      src/test/java/...
  agents/
  valid_classes.json            # optional, generated by dataset_parser
```

The iterative evolution flow expects projects under `dataset/<project_name>` and writes outputs under directories such as `test_reports/`, `evolution_process/`, and `historical_best/`.

## Prerequisites

- Python environment with the project dependencies installed.
- Java and Maven available on `PATH`.
- Optional: set `MAVEN_CMD` if Maven is not discoverable from `PATH`.
- LLM/API configuration required by the generator and repair modules.
- For iterative evolution, coverage reports are produced from Maven/JaCoCo/Surefire outputs.

## 0. Discover Target Classes

Use the dataset parser to scan Java projects and build class metadata.

```bash
# Scan dataset/ under the current working directory
python agents/dataset_parser.py

# Specify a dataset root
python agents/dataset_parser.py --dataset C:/path/to/dataset

# Disable Tree-sitter advanced analysis
python agents/dataset_parser.py --disable-tree-sitter

# Enable verbose debug output
python agents/dataset_parser.py --debug
```

## 1. Generate Initial Tests

```bash
# Test API connectivity
python -m agents.test_generator --test-api

# Generate tests for all detected projects/classes
python -m agents.test_generator

# Generate tests for one project
python -m agents.test_generator --project <project_name>

# Generate tests for one class
python -m agents.test_generator --project <project_name> --class <ClassName>

# Enable parallel generation
python -m agents.test_generator --project <project_name> --parallel --workers 8

# Analyze class/test distribution without generation
python -m agents.test_generator --project <project_name> --analyze-only
```

Useful options include `--count`, `--verify-method-count`, `--regenerate-insufficient`, and `--force-rename-classes`.

## 2. Repair Tests

`test_generator` can call repair automatically after generation failures. You can also use `agents/test_repair` directly from Python for targeted repair workflows; see `agents/test_repair/README.md`.

Repair flow:

```text
Rule-based repair -> LLM repair -> deletion/fallback handling
```

## 3. Run Iterative Evolution

```bash
# Basic run
python -m agents.iterative_evolution --base_dir . --project <project_name> --max_gen 10

# Target a single class
python -m agents.iterative_evolution --base_dir . --project <project_name> --target_class <ClassName> --max_gen 10

# Run a generation range
python -m agents.iterative_evolution --base_dir . --project <project_name> --start_gen 2 --end_gen 10

# Force regeneration/overwrite of existing reports in the selected range
python -m agents.iterative_evolution --base_dir . --project <project_name> --start_gen 2 --end_gen 10 --force_overwrite

# Adjust coverage/fitness settings
python -m agents.iterative_evolution --base_dir . --project <project_name> --max_gen 8 --fitness_threshold 0.95 --branch_target 95.0
```

## Common Full Flow

```bash
# 1. Parse dataset metadata
python agents/dataset_parser.py --dataset ./dataset

# 2. Generate initial tests
python -m agents.test_generator --project <project_name> --parallel --workers 8

# 3. Evolve generated tests
python -m agents.iterative_evolution --base_dir . --project <project_name> --max_gen 10
```

## Outputs

Typical generated outputs include:

- `dataset/<project_name>/src/test/java/...`: generated or evolved test files.
- `test_reports/<project_name>/Gen<N>/...`: coverage and test execution reports.
- `evolution_process/<project_name>/Gen<N>/...`: per-generation test snapshots.
- `historical_best/<project_name>/...`: best tests saved across generations.
- `generation_statistics/` or module-specific statistics directories when enabled.

## Notes

- Keep project names and target class names as runtime inputs; the code should not depend on hardcoded repositories or classes.
- If Maven cannot be found, set `MAVEN_CMD` to the Maven executable path.
- Review module READMEs for more detailed configuration and troubleshooting.
